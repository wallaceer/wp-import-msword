    Tipologia Pagina (alfanumerico, tratta o porto) - campo acf
    Titolo pagina - campo wp
    slug - campo wp
    Testo pagine (con eventualmente titoli interni al testo marcati singolarmente come ci siamo detti) - campo wp
    Pagina genitore - campo wp
    Template - campo wp
    Title tag - campo toast
    Meta description - campo toast
    Immagine (solo nome file) - campo acf
    Macroarea (alfanumerico credo di un carattere) - campo acf
    from (alfanumerico credo di 4 caratteri) - campo acf
    to (alfanumerico credo di 4 caratteri) - campo acf => VUOTO x Pagine porto
    A/r (campo che indica se la tratta prevede entrambe le direzioni, generalmente è così quindi sarà un default = true) - campo acf => VUOTO x Pagine porto


    I campi macroarea, from e to servirebbero in fase di fase di valorizzazione dell’HTML per estrarre dal db le pagine correlate.
    Ti faccio degli esempi:

        In una home di macroarea (es. Croazia) tutte le pagine tratte associate che hanno il campo macroarea con la stessa valorizzazione della pagina genitore (la Croazia, anche lei avrebbe questo custom field)
        In una pagina porto tutte le pagine tratte associate che hanno il campo macroarea e il campo from con la stessa valorizzazione della pagina porto
        In una pagina tratta tutte le (2) pagine porto associate che hanno il campo macroarea e i campi from e to con la stessa valorizzazione della pagina tratta


    In pratica con i campi campi macroarea, from e to si gestirebbe buona parte della navigazione all’interno del sito.

I campi ACF dovrebbero essere questi:

    Tipologia pagina, alfanumerico (da confermare durante la progettazione del Backend perché potremmo farne a meno)
    Immagine, alfanumerico (conterrà il nome del file, es. immagine.jpg, senza doverne gestire l’upload)
    Macroarea, alfanumerico di 1 carattere (è casesensitive, il valore deve essere sempre maiuscolo )
    From, alfanumerico di 4 caratteri (è casesensitive, il valore deve essere sempre maiuscolo )
    to, alfanumerico di 4 caratteri (è casesensitive, il valore deve essere sempre maiuscolo )
    A/r, option di default sempre true
